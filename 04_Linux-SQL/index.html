                the Hardware returns information to the OS and the OS then returns to the Application     
        
        02) Requests to the Operating System
                - First, the user decides they want to download a file that they found online, so they click on a download button near the file in the internet browser application.
                - Then, the internet browser communicates this action to the OS.
                - The OS sends the request to download the file to the appropriate hardware for processing.
                - The hardware begins downloading the file, and the OS sends this information to the internet browser application. The internet browser then informs the user when the file has been downloaded.

04.12 Authenticate and Authorize Users

        01) File Permissions and Ownership
            Permissions - type of access granted for a file or directory
            Authorization - granting specific access to specific resoureces in a system
                   allows you to limit access to files or directories 
            3 Linux Permissions:
                    - Read - can read a file, or all files in a directory
                    - Write - allow content to be modified in a file, can create and modify files in a directory
                    - Execute - allow files to be executed (if executable), can enter into directories and alter files 
            3 Linux Owners:
                    - Users - the owner of the file
                    - Group - consists of several users, 
                    - Other - anyone else on the system that isn't the user or part of the group

        File Permissions - ex: drwxrwxrwx  = a directory with full permissions for the user group 
            first 'd' = directory
              1st 'rwx' = read, write, execute -- permissions for the user
              2nd 'rwx' = permissions for the 'group'
              3rd 'rwx' = permissions for 'other'

           Options: modify the behavior of the command

           ls -la  = show permissions of all files (including hidden)
        
        02) Change Permissions
               chmod = changes permissions of files and directories (change mode)
                 ex: chmod g+w, o-r access.txt       - owner types seperated by comma
                    this will add 'write' permission to 'group' 
                          and remove the 'read' permission from 'other'
        
        03) Permission Commands
                read - file = read, directory = read contents for directory and file 
                write - file = write to file, directory = write to files in the directory
                execute - file = execute command files (executables), directory = enter directory and access files

               1st letter / - = directory or file
               2nd group of 3 letters = read, write, execute for 'user' == a hyphen = no permission
               3rd group of 3 letters = read, write, execute for 'group'
               4th group of 3 letters = ready, write, execute for 'other'
               
               ls -    = list
               ls -a   = list with hidden files
               ls -l   = list with permissions
               ls -la  = list with permissions and hidden files
               
            principle of least privlege = limit access to files and directories, unless access is required   

            chmod = change mode -- allows user to change permissions for files and directories in linux

            ex: chmod u+rwx, g+rwx, o-rwx login_session.txt
                  this would grant 'user' - read, write, execute file permission
                       would grant 'group' - read, write, execute file permission
                       and would remove 'other' -read, write, execute file permission

        04) Filters AND, OR, and NOT
                AND - Multiple conditions must be met, specifies that both conditions must be met simultaneously.
                  ex: SELECT * FROM machines WHERE operating_system = 'OS 1' AND email_client = 'Email Client 1';

                OR  - either condition can be met
                  ex: SELECT * FROM machines WHERE operating_system = 'OS 1' OR operating_system = 'OS 3';  
                
                NOT - negates a condition
                  ex: SELECT * FROM machines WHERE NOT operating_system = 'OS 3';

        05) More on Filters with AND, OR, and NOT
                AND - filters 2 conditions and specifies that both are met
                   ex: SELECT firstname, lastname, email, country, supportrepid FROM customers WHERE supportrepid = 5 AND country = 'USA';

                OR - filters 2 conditions and specifies either can be met
                   ex: SELECT firstname, lastname, email, country
                        FROM customers
                        WHERE country = 'Canada' OR country = 'USA'; 

                NOT - only works on a single condition -- this operator negates a condition, SQL returns records that DO NOT match the conditions of query
                  <> & != are both the same operator as NOT -- the following examples give the same results 
                   ex: WHERE NOT country = 'USA';
                   ex02: WHERE country <> 'USA';
                   ex03: WHERE country != 'USA';

                Combining Logical Operators:

                can use AND and NOT together-
                   ex: WHERE NOT country = 'Canada' AND NOT country = 'USA';
                   ex02: WHERE coutry != 'Canada' AND country != 'USA';

        03) Compare Types of Joins
                INNER JOIN: only returns rows where there is a match 
                    ex: SELECT * FROM employees INNER JOIN machines ON employees.device_id = machines.device_id;
                LEFT JOIN: returns all of 1st table and matching rows from 2nd table
                    ex: SELECT * FROM employees LEFT JOIN machines ON employees.device_id = machines.device_id;
                RIGHT JOIN: returns all of 2nd table and matching rows from 1st 
                    ex: SELECT * FROM employees RIGHT JOIN machines ON employees.device_id = machines.device_id;
                FULL OUTER JOIN: returns all records from both tables
                    ex: SELECT * FROM employees FULL OUTER JOIN machines ON employees.device_id = machine.device_id;
