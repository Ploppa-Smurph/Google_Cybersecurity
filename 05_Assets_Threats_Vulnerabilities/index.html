    02) The What, Why, and How of Asset Security
            Risk - anything that can impact the confidentiality, integrity, or availability of an asset
                    CIA Triad = confidentiality, integrity, availability
            security risk planning is made based on 3 components: assets, threats, vulnerabilities
                - Asset - item perceived as having value to organization; can be physical or digital 
                - Threat - circumstance or event that can negatively affect assets; ex: a thief, bad storm, fire
                - Vulnerability - a weakness that can be exploited; ex: an unlocked door to a restricted area, hole in roof letting rain in

    07) Activity: Classify the Assets Connected to a Home Network
            One of the most valuable assets in the world today is information. Most information is accessed over a network. There tend to be a variety of devices connected to a network and each is a potential entry point to other assets.
        
            An inventory of network devices can be a useful asset management tool. An inventory can highlight sensitive assets that require extra protection.
    
            A home office that includes a desktop computer, printer, router, webcam, speakers, headphones, and external hard drive.
    
            You’re operating a small business from your home and must create an inventory of your network devices. This will help you determine which ones contain sensitive information that require extra protection.
    
    
            To do this, you will start by identifying three devices that have access to your home network. This might include devices such as:
               - Desktop or laptop computers
               - Smartphones
               - Smart home devices

               - Video streaming devices
    
        Then, you’ll list important characteristics of each device such as its owner, location, and type. Finally, you will assign each device a level of sensitivity based on how important it is to protect.

                1st NIST CSF Core function is Identify - Asset management and risk assessment relate to 1. Identify

    04) Activity: Score Risks Based on Their Likelihood and Severity	
            You might recall that the purpose of having a security plan is to be prepared for risks. Assessing potential risks is one of the first steps of the NIST Cybersecurity Framework (CSF), a voluntary framework that consists of standards, guidelines, and best practices to manage cybersecurity risk. Risk assessments are how security teams determine whether their security operations are adequately positioned to prevent cyber attacks and protect sensitive information.

            You've joined a new cybersecurity team at a commercial bank. The team is conducting a risk assessment of the bank's current operational environment. As part of the assessment, they are creating a risk register to help them focus on securing the most vulnerable risks.

            A risk register is a central record of potential risks to an organization's assets, information systems, and data. Security teams commonly use risk registers when conducting a risk assessment.

            Your supervisor asks you to evaluate a set of risks that the cybersecurity team has recorded in the risk register. For each risk, you will first determine how likely that risk is to occur. Then, you will determine how severely that risk may impact the bank. Finally, you will calculate a score for the severity of that risk. You will then compare scores across all risks so your team can determine how to prioritize their attention for each risk.

    02) Security Controls
            Security Controls = safeguards designed to reduce specific security risks

            3 types of Security Controls:
                 - Technical: technologies used to protect assets. ex: encryption, authentication systems, etc...
                 - Operational: relate to maintaining day-to-day security environment. people perform these controls. ex: awareness training and incident response
                 - Managerial: centered around how technical and operational controls reduce risk. ex: policies, standards, procedures 

            Information Privacy - protection of unauthorized access and distribution of data      

            security controls are intended to limit access based on user and situation
              Principle of Least Privilege

            Data Owner = person that decides who can access, edit, use, or destroy information
            Data Custodian = anyone or anything that's responsible for the safe handling, transport, and storage of information

            data is an asset and requires proper classification and handling
 
    03) Principle of Least Privilege
            Principle of Least Privilege - limit access to reduce risk 

            determine access and authorization 
               common user accounts: 
                        - Guest account: provided to allow access to customes, clients, contractors, or business partners
                        - User account: assigned based on job duties
                        - Service account: granted to applications that need to interact with other software on the network
                        - Privileged account: elevated permissions or administrative access

        3 types of common user account audits: 
          - Usage audits: review with resources each account is accessing. 
                 determine whether users are acting in accordance with an organization's security policies
          - Privilege audits: assess whether user's role is in alignment with the resources they have access to
                 determine if access privileges are accumulated (check for 'privilege creep') 
          - Account change audits: looks for suspicious activity like multiple attempts to change password
                 determine if account changes are made by authorized users

05.07 Encryption Methods

    01) Fundamentals of Cryptography
            PII (Personally Identifiable Information) - any information that can be used to infer an individual's identity

            Cryptography - process of transforming information into a form that unintended readers can't understand 
              encrypt data to hide information, decrypt data to unhide the data
             
            ciphertext - the encrypted / unreadable text
            plaintext - clearly readable text

            Algorithm = set of rules that solve a problem
            Cipher = algorithm that encrypts information

            Cryptographic Key = mechanism that decrypts ciphertext

            Brute Force Attack = trial and error process of discovering private information

    02) Public Key Infrastructure
            PKI (Public Key Infrastructure) = encryption framework that secures exchange of information online 

            Asymmetric Encryption = uses public and private key pair for encryption and decryption of data
            Symmetric Encryption = uses a single secret key to exchange information; faster and LESS secure 

            PKI 1: exchanges encrypted information 
                2: establish trust using a system of digital certificates
               
            Digital Certificate = file that verifies the identity of a public key holder    

                In PKI, data can be encrypted using asymmetric encryption, symmetric encryption, or both. 
                Then, a digital certificate binds the data's public key to the verified identity of a website, individual, organization, device, or server.   

             1. Certificate Authority recieves basic information about company and website   
             2. Certificate Authority uses public key and organization's basic information to confirm company's identity
             3. When the information is confirmed the Certificate Authority encrypts the data with the CA's Private Key
             4. the Digital Certificate is created that contains encrypted company data and the CA's digital signature to prove authenticity

    03) Symmetric and Asymmetric Encryption
            Encryption = process of converting data from a readable format to an encoded format
            Public Key Infrastructure (PKI) = encryption framework that secures exchange of online information
            Cipher = algorithm that encrypts information

            Symmetric encryption = single secret key to exchange information 
            Asymmetric encryption = public and private key pair for encrypting and decrypting data
               public key -> encrypts the data
               private key -> decrypts the data  

         ciphers are vulnerable to Brute Force Attacks -- make a long and complex key 

         Approved Algorithms: 
             Symmetric Algorithms - 
                - Triple DES (3DES) - known as a block cipher. converts plaintext into ciphertext in 'blocks'
                   developed in the 1970s. 3DES = 192 bit keys
                - Advanced Encryption Standard (AES) - AES generates keys that are 128, 192, or 256 bit. 
                
             Asymmetric Algorithms -
                - Rivest Shamir Adleman (RSA) - named for the 3 creators
                  used to protect Highly Sensitive Data - uses keys of 1024, 2048, or 4096 bits!
                - Digital Signature Algorithm (DSA) - introduced by NIST in 1990s. key lenght of 2048 bits.      

         Generating Keys: OpenSSL -- can be used to generate public and private keys
         
         ** Heartbleed Bug ** -- vulnerability in OpenSSL, exposed sensitive data in the memory of sites and applications. 
                demonstrated the importance of patching

        Obscurity is NOT Security  = a cryptographic system should NOT be considered secure if it requres secrecy around how it works         

        many websites use both Asymmetric and Symmetric encryption. Asymmetric is when you first login and verify username and password,
          then when user is verified the rest of the communication switches to Symmetric encryption for quicker communication.

    04) Activity: Decrypt an Encrypted Message
        You have multiple tasks in this lab:
          - List the contents of a directory
          - Read the contents of files
          - Use Linux commands to revert a classical cipher back to plaintext
          - Decrypt an encrypted file and restore the file to its original state

          In this scenario, all of the files in your home directory have been encrypted. You’ll need to use Linux commands to break the Caesar cipher and decrypt the files so that you can read the hidden messages they contain.

            Here’s how you’ll do this task: First, you’ll explore the contents of the home directory and read the contents of a file. Next, you’ll find a hidden file and decrypt the Caesar cipher it contains. 
              Finally, you’ll decrypt the encrypted data file to recover your data and reveal the hidden message.

              cat .leftShift3 | tr "d-za-cD-ZA-C" "a-zA-Z"  
                 - The tr command translates text from one set of characters to another, using a mapping. The first parameter to the tr command represents the input set of characters, and the second represents the output set of characters. 
                   Hence, if you provide parameters “abcd” and “pqrs”, and the input string to the tr command is “ac”, the output string will be “pr".

              openssl aes-256-cbc -pbkdf2 -a -d -in Q1.encrypted -out Q1.recovered -k ettubrute     
                  - In this instance, the openssl command reverses the encryption of the file with a secure symmetric cipher, as indicated by AES-256-CBC. The -pbkdf2 option is used to add extra security to the key, and -a indicates the desired encoding for the output. 
                    The -d indicates decrypting, while -in specifies the input file and -out specifies the output file. The -k specifies the password, which in this example is ettubrute.
