03.03 Network Communication

        01) Introduction to Network Communication
                  Communication - sending data packets from one point to another
                  Data Packets - have information about sender, receiver, and content of message
                   Header - contains IP address and MAC address of destination device, also contains "Protocol Number" that tells receiving device what to do with the information
                   Body - contains data that needs to be transmitted during communication
                   Footer - tells the receiving device that the communication is complete
                
                Network Performance is measured by 'Bandwidth'
                 Bandwidth = amount of data a device receives every second 
                
                Packet Sniffing - practice of capturing and inspecting data packets sent across the network

        02) TCP/IP Model
                Transmission Control Protocol - internet communication protocol that allows 2 devices to connect and stream data 
                Internet Protocol - set of standards used for routing and addressing data packets as they travel between devices on the internet
                Port - software-based location that organizes the sending and receiving of data between devices on a network
                    Port 25 - email communication
                    Port 443 - Secure Internet communication (HTTPS)
                    Port 20 - Large File Transfers
                
        03) 4 layers of TCP/IP
                TCP/IP model is a framework used to visualize how data is organized and transmitted across the network
                   1. Network Access Layer - deals with the creation of data packets and their transmission across the network
                            - includes hardware devices                   
                   2. Internet Layer - where IP address is attactched to indicate location of sender and receiver. 
                            - focus on how networks communicate - handles LAN communication or sending out to WAN
                   3. Transport Layer - controls flow of traffic across the network. permit or deny communications with other devices and includes connection status information
                            - error control / handling
                   4. Application Layer - protocols determine how data packets will interact with the receiving device
                            - functions organized at application layer include file transfers and email services
        
        04) Learn More about TCP/IP
                1 - Network Access Layer = Ethernet vs Wireless (and more)
                2 - Internet Layer = IP (v4, v6)
                3 - Transport Layer = TCP v UDP (and more)
                4 - Application Layer = HTTP, TLS, DNS, etc... 

                Network Access Layer (aka Data Link Layer) - hubs, modems, cables and wiring are part of this layer
                        - ARP (Address Resolution Protocol) is part of the Network Access Layer
                           ARP assists IP with directing data packets on the same physical network by mapping IP addresses to MAC addresses.
                Internet Layer (aka Network Layer) - responsible for ensuirng delivery to destination. 
                        - IP (Internet Protocol) - sends data packets to the correct destination. IP packets allow communication between two networks
                        - ICMP (Internet Control Message Protocol) - shares information and status updates of data packets. useful for detecting and troubleshooting network errors.
                                ICMP reports information about packets that were dropped 
                Transport Layer - responsible for transmitting data to destination.
                        - Transmission Control Protocol (TCP) - ensures data is reliably transmitted to the destination
                        - User Datagram Protocol (UDP) - not concerned with reliability of transmission - performance sensitive applications like video streaming
                Application Layer - similar to Application, Presentation, and Session layers of OSI model. responsible for making network requests and responding to requests. 
                    defines which internet services and applications users can access. 
                         - Hypertext Transfer Protocol (HTTP) 
                         - Simple Mail Transfer Protocol (SMTP)
                         - Secure Shell (SSH)
                         - File Transfer Protocol (FTP)
                         - Domain Name System (DNS)
                TCP/IP combines multiple OSI model layers into each layer (TCP/IP has 4 layers, OSI has 7 Layers)                               

        05) the OSI (Open Systems Interconnection) Model
                the TCP/IP model is a framework used to visualize how data is organized and transmitted across networks.

            the OSI model is a standerdized concept that describes the 7 layers computers use to communicate and send data over the network
                1. Physical Layer - corresponds to physical hardware. Hubs, Modems, Cables and Wiring 
                2. Data Link Layer - sending and receiving data packets within a single network - Switches and NIC (Network Interface Cards) on LANs
                3. Network Layer - oversees receiving frames from data link (layer 2) and deliving to intended destination
                        ex: Data Packets allow communication between 2 networks.
                4. Transport Layer - responsible for delivering data between devices. handles data transfer, flow and breaking data down to smaller data packets
                        ex: TCP and UPD are transport layer protocols
                5. Session Layer - describes when connection is established between 2 devices.  
                        ex: authentication, reconnection, setting checkpoints of data transfer
                6. Presentation Layer - Data translation and encryption for the network, processes at the presentation layer require a standardized format.
                        ex: encryption, compression, etc... take place. 1 example of encryption takes place at this layer is SSL to encrypt data for HTTPS
                7. Application Layer - processes directly involving everyday users; 'user connection to the network via applications and request'
                        ex: browsers utilize HTTP / HTTPS and also DNS to translate web domain names to IP addresses; email uses SMTP (Simple Mail Transfer Protocol) to send and receive email

        02) Components for Network Layer Communications
                IPv6 data packet header is simpler than IPv4 
                    13 parts of IPv4 header:
                        Version: The first 4-bit header tells receiving devices what protocol the packet is using. The packet used in the illustration above is an IPv4 packet.
                        IP Header Length (HLEN): HLEN is the packet’s header length. This value indicates where the packet header ends and the data segment begins. 
                        Type of Service (ToS): Routers prioritize packets for delivery to maintain quality of service on the network. The ToS field provides the router with this information.
                        Total Length: This field communicates the total length of the entire IP packet, including the header and data. The maximum size of an IPv4 packet is 65,535 bytes.
                        Identification: For IPv4 packets that are larger than 65, 535 bytes, the packets are divided, or fragmented, into smaller IP packets. The identification field provides a unique identifier for all the fragments of the original IP packet so that they can be reassembled once they reach their destination.                     
                        Flags: This field provides the routing device with more information about whether the original packet has been fragmented and if there are more fragments in transit.
                        Fragmentation Offset: The fragment offset field tells routing devices where in the original packet the fragment belongs.
                        Time to Live (TTL): TTL prevents data packets from being forwarded by routers indefinitely. It contains a counter that is set by the source. The counter is decremented by one as it passes through each router along its path. When the TTL counter reaches zero, the router currently holding the packet will discard the packet and return an ICMP Time Exceeded error message to the sender. 
                        Protocol: The protocol field tells the receiving device which protocol will be used for the data portion of the packet.
                        Header Checksum: The header checksum field contains a checksum that can be used to detect corruption of the IP header in transit. Corrupted packets are discarded.
                        Source IP Address: The source IP address is the IPv4 address of the sending device.
                        Destination IP Address: The destination IP address is the IPv4 address of the destination device.
                        Options: The options field allows for security options to be applied to the packet if the HLEN value is greater than five. The field communicates these options to the routing devices.

        02) Network Protocols
                Network Protocols are a set of rules used by 2 or more devices on a network to describe the order of delivery and structure of data   
                ex: a) when accessing a website, your device first uses TCP to establish a connection to the web server.
                     -TCP allows 2 devices to connect and stream data, TCP also verifies both devices (handshake)
                    b) the connection is established after the web server responds and sends data packets. the ARP (Address Resolution Protocol) helps get the data to the correct MAC address
                    c) now we can access the website, HTTPS allows webbrowsers to more securely send and receive data across the internet
                      -HTTPS uses SSL/TLS for a Secure Socket Layer and Transport Layer Security to encrypt data and send securely across the internet
                    d) DNS or Domain Name Server translates the IP address to a web address

        03) Common Network Protocols
                Communication Protocols
                    - TCP (Transmission Control Protocol): allows 2 devices to connect. (happens at Transport Layer of TCP/IP model)
                        3 way handshake = device sends SYN request -> server responds with SYN/ACK to acknowledge -> then TCP connection is established 
                    - UDP (User Datagram Protocol): connectionless protocol, less reliable than TCP. UPD happens at Transport Layer
                    - HTTP (Hypertext Transfer Protocol): Application Layer Protocol - uses Port 80. communicate between browser and server
                    - DNS (Domain Name Server): translates IP address into domain name. DNS occurs at Application layer -- uses UDP & port 53 (default)
                Management Protocols
                    - SNMP (Simple Network Management Protocol): monitor and manage devices on internet - SNMP occurs at Application Layer
                    - ICMP (Internet Control Message Protocol): used to troubleshoot network connectivity isses, ICMP occurs at Internet Layer
                Security Protocols
                    - HTTPS (Hypertext Protocol Secure): Port 443; Application Layer.
                    - SFTP (Secure File Transfer Protocol): TCP port 22 (SSH); SFTP uses AES (Advanced Encryption Standar). SFTP occurs at Application Layer

        02) Virtual Private Networks (VPNs)

        03) Security Zones
                segment of the network that protects the internal network from the internet
                Network Segmentation: security technique that divides network into segments each with their own access permissions and security rules
                  ex: guest wi-fi at coffee shop or hotel, there is a seperate password protected connection for business operations 
                Subnets - maintains privacy for different departments in a domain  

                Uncontolled Security Zone: all networks outside of an organization's control
                Controlled Security Zone: area to protect the internal network
                   DMZ (Demilitarized Zone): Contains Web-servers, Proxy servers, DNS servers, E-mail, File Servers for external communication
                   Internal Network: Private data the organization needs to protect
                   Restricted Zone: only accessible to certain employees in the internal network, more secure
                should have DMZ between 2 firewalls - 1st filters traffic outside DMZ, 2nd filters traffic moving into internal network 
                    there would be an additional firewall between the internal network and restricted zone
